What this prototype does

This is a Retrieval-Augmented Generation (RAG) prototype built on public OOCL workflow documentation. When you ask a question, it retrieves the most relevant source chunks, then generates an answer using only that context.

Source documents
  • B/L Document Manager (oocl.com)
  • Shipping Instructions (oocl.com)
  • USA Import Procedures (oocl.com)
  • B/L Amendment Request guide (OOCL MOC PDF)
Key design decisions
  • Answers are grounded in source documents only — no hallucination
  • Source chunks are shown with every answer for verification
  • If the source doesn't contain the answer, the assistant says so
  • User feedback (helpful / not helpful) is collected per answer
  • Evaluation tab tests all 6 scenarios including hallucination resistance
Tech stack
  • Backend: Python, FastAPI
  • Vector search: ChromaDB with sentence-transformers embeddings
  • Answer generation: OpenAI GPT-4o-mini
  • Frontend: Plain HTML/CSS/JS (no framework)
